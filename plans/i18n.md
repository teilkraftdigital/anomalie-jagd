# Plan: i18n (v0.5.0)

> Quelle: Grill-Me-Session 2026-04-02

## Architekturentscheidungen

- **Sprachen**: Deutsch (DE) + Englisch (EN) — weitere Sprachen nicht geplant
- **Library**: `react-i18next` — de-facto Standard, native Namespace-Unterstützung, minimale API
- **Sprachumschalter**: Toggle-Button auf der Startseite + Persistenz in `localStorage`; Browser `Accept-Language` als Fallback beim ersten Besuch
- **Namespaces**: `common` (App-weite UI) + `scene-{id}` pro Scene (z. B. `scene-button`, `scene-form`)
- **Scene-Locales**: Co-located in `src/app/scenes/{id}/locale/` — Scene ist geschlossene Einheit
- **Trennung Domain / i18n**: `Scene<TModel>` und `Patch<TModel>` wissen nichts von i18n. Locales werden separat exportiert und nur in `App.tsx` beim Registrieren eingespielt
- **Registrierung**: `import { scene, locales } from "./app/scenes/button"` → `registerScene(scene)` + i18next Namespace registrieren
- **Patch-Typ**: `scene: string` (z. B. `"form"`) + `id: string` (z. B. `"patch-no-label"`) als separate Felder; `label` und `explanation` entfernt — leben nur noch in Locales
- **Locale-Key**: `${patch.scene}-${patch.id}` = `"form-patch-no-label"` — identisch mit bisherigem `patch.id`, Store/`discoveredPatchIds` bleibt kompatibel
- **TypeScript-Enforcement**: Jede Scene hat `locale/types.ts` der `PatchId` aus dem Patches-Array ableitet — fehlt ein Locale-Eintrag für einen Patch → Compile-Fehler
- **Scene-Typ**: `Scene.name` und `Scene.description` entfernt — nur noch in Locales
- **DIFFICULTIES**: `label` und `description` aus `DIFFICULTIES`-Konstante entfernt → `common.difficulty`

### TypeScript-Enforcement (`src/app/scenes/button/locale/types.ts`)

```ts
import { patches } from "../patches";

type PatchId = (typeof patches)[number]["id"];

export type ButtonSceneLocale = {
  scene: { name: string; description: string };
  renderer: { label: string };
  patches: Record<PatchId, { label: string; explanation: string }>;
};
```

## Phase 1: Setup + common

### Was zu bauen ist

`react-i18next` und `i18next` installieren. `src/i18n/` anlegen mit `config.ts` (Initialisierung, localStorage-Detection, Fallback `de`) sowie `locale/de.ts` und `locale/en.ts` für den `common`-Namespace. App mit `I18nextProvider` wrappen. Alle App-weiten UI-Strings in Komponenten und Pages durch `t("common:...")` ersetzen. Sprachumschalter (DE/EN-Toggle) auf der Startseite einbauen.

### Akzeptanzkriterien

- [ ] `react-i18next` und `i18next` installiert
- [ ] `src/i18n/config.ts` initialisiert i18next mit `common`-Namespace, `lng: localStorage || navigator.language || "de"`, Fallback `de`
- [ ] `locale/de.ts` und `locale/en.ts` für `common` vollständig
- [ ] `App.tsx` mit `I18nextProvider` gewrappt
- [ ] Alle Strings in `Toolbar`, `GameShell`, `RoundDisplay`, Pages durch `t()` ersetzt
- [ ] `DIFFICULTIES` — `label`/`description` aus Konstante entfernt, Werte in `common.difficulty`
- [ ] Sprachumschalter auf Startseite: Toggle DE/EN, persistiert in `localStorage`
- [ ] `npm run build`, `npm run lint`, `npm test` fehlerfrei

## Phase 2: Domain-Typen bereinigen

### Was zu bauen ist

`Scene<TModel>`: `name` und `description` entfernen. `Patch<TModel>`: `label` und `explanation` entfernen, `scene: string` als neues Pflichtfeld hinzufügen. Alle bestehenden Patch-Dateien um `scene`-Feld ergänzen (Button-Patches: `scene: "button"`, Form-Patches: `scene: "form"`). `discoveredPatchIds` im Store bleibt kompatibel — Canonical Key `${patch.scene}-${patch.id}` entspricht dem bisherigen `patch.id`.

### Akzeptanzkriterien

- [ ] `Scene<TModel>` ohne `name` und `description`
- [ ] `Patch<TModel>` ohne `label` und `explanation`, mit `scene: string`
- [ ] Alle Button-Patches: `scene: "button"` ergänzt
- [ ] Alle Form-Patches: `scene: "form"` ergänzt
- [ ] `${patch.scene}-${patch.id}` ergibt denselben Wert wie bisher `patch.id`
- [ ] Store, Pool-Logik, Discovery-Logik kompilieren fehlerfrei
- [ ] `npm run build`, `npm run lint`, `npm test` fehlerfrei

## Phase 3: Button-Scene i18n

### Was zu bauen ist

`src/app/scenes/button/locale/` anlegen mit `types.ts`, `de.ts`, `en.ts`. `types.ts` leitet `PatchId` aus dem Patches-Array ab und definiert `ButtonSceneLocale`. `de.ts` und `en.ts` werden gegen den Typ geprüft — Compile-Fehler bei fehlendem Patch-Eintrag. Locales aus `index.ts` als `locales` exportieren. `App.tsx`: `registerScene` + i18next-Namespace für Button-Scene registrieren. Button-Renderer durch `t()` ersetzen.

### Akzeptanzkriterien

- [ ] `locale/types.ts` mit `ButtonSceneLocale` und `PatchId`-Ableitung
- [ ] `locale/de.ts` und `locale/en.ts` vollständig und typgeprüft
- [ ] `index.ts` exportiert `{ buttonScene, locales }`
- [ ] `App.tsx` registriert Button-Scene-Namespace in i18next
- [ ] Button-Renderer nutzt `t("scene-button:renderer.label")`
- [ ] Glossar zeigt `t("patches.patch-div-button.label", { ns: "scene-button" })`
- [ ] `npm run build`, `npm run lint`, `npm test` fehlerfrei

## Phase 4: Form-Scene i18n

### Was zu bauen ist

Analog zu Phase 3 für die Form-Scene. Der Form-Renderer hat deutlich mehr Strings: Feld-Labels, Placeholder, Hint-Texte, Validierungs-Fehlermeldungen, Submit-Button, `required-note`. Alle diese Strings wandern in `locale/de.ts` und `locale/en.ts` unter `renderer.fields.{fieldId}.*`.

### Akzeptanzkriterien

- [ ] `locale/types.ts` mit `FormSceneLocale` — alle Renderer-Strings typisiert
- [ ] `locale/de.ts` und `locale/en.ts` vollständig inkl. Validierungsmeldungen
- [ ] Form-Renderer nutzt `t()` für alle Strings (Labels, Placeholder, Hints, Fehler, Submit)
- [ ] `required-note`-Text aus Locale
- [ ] `index.ts` exportiert `{ formScene, locales }`
- [ ] `App.tsx` registriert Form-Scene-Namespace
- [ ] `npm run build`, `npm run lint`, `npm test` fehlerfrei
- [ ] CHANGELOG.md für v0.5.0 aktualisiert

## Dateistruktur (Zielbild)

```
src/
  i18n/
    config.ts
    locale/
      de.ts          ← common
      en.ts          ← common
  app/
    scenes/
      button/
        locale/
          types.ts   ← ButtonSceneLocale, PatchId
          de.ts
          en.ts
        index.ts     ← export { buttonScene, locales }
      form/
        locale/
          types.ts   ← FormSceneLocale, PatchId
          de.ts
          en.ts
        index.ts     ← export { formScene, locales }
```

## App.tsx (Zielbild)

```ts
import { buttonScene, locales as buttonLocales } from "./app/scenes/button";
import { formScene, locales as formLocales } from "./app/scenes/form";

// i18n Namespaces registrieren
i18n.addResourceBundle("de", "scene-button", buttonLocales.de);
i18n.addResourceBundle("en", "scene-button", buttonLocales.en);
i18n.addResourceBundle("de", "scene-form", formLocales.de);
i18n.addResourceBundle("en", "scene-form", formLocales.en);
```
